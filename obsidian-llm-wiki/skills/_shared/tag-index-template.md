## Tag Usage Guidelines

### When Creating Tasks
1. **Start with work area**: main project or domain tag
2. **Add technical area**: technology or platform tag
3. **Add work type**: `#review`, `#demo`, `#doc`, etc.
4. **Add people if relevant**: person tags
5. **Special status last**: `#hide`, `#blocked`, etc.

### Example Task Tags
```markdown
# Documentation task for a project
#doc #projectA #report

# Review task
#review #projectA #report

# Personal Obsidian task (hidden from work dashboards)
#obsidian #personal #hide
```

## Adding New Tags

New tags should only be added when:
1. **Unique value**: Provides distinct classification not covered by existing tags
2. **Dashboard potential**: Could be useful for filtering/querying in dashboards
3. **User approval**: Always get user confirmation before using new tags
4. **Document here**: Add to this index once approved
