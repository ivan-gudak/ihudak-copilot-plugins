# Dynatrace terminology

Rules for Dynatrace product names, solutions, trademarks, and branded terms.
Source: styleguide.dynatrace.com/docs/dynatrace-terminology, /docs/dynatrace-trademarks

---

## Registered trademarks

The following terms carry the ® symbol. Use ® on **first mention per page/document**
only. Never use ® in headings, titles, or pluralised forms. Never make a trademarked
term possessive when the ® symbol is attached.

- Dynatrace®
- OneAgent®
- PurePath®
- Smartscape® real-time dependency graph
- Grail® unified data lakehouse
- OpenPipeline®
- DevOne®

### ® placement rules

- ✅ Dynatrace® provides end-to-end observability.  (first mention)
- ✅ Dynatrace provides end-to-end observability.    (subsequent mentions)
- ❌ Dynatrace®'s platform  (never possessive with ®)
- ✅ The Dynatrace® platform  (use as adjective instead)
- ❌ Use multiple OneAgents®  (never pluralise with ®)
- ✅ Use multiple OneAgent® instances

### Third-party trademarks

When mentioning third-party products, use their official capitalisation and
trademark symbols on first mention. Never alter their names.

- ✅ Kubernetes, Amazon Web Services (AWS), Microsoft Azure
- ❌ kubernetes, amazon web services, AZURE

## Apps terminology

### App types
- **Built-in apps** — provided by default (Notebooks, Workflows, Dashboards)
- **Classic apps** — Dynatrace Classic capabilities as apps (Security Overview, etc.)
- **Custom apps** — customer/partner-created
- **On-demand apps** — optional, installed from Hub (Carbon Impact, SRG, etc.)
- **Sample apps** — example apps for developers

### App naming
- ✅ Open Dashboards to get started.
- ❌ Open the Dashboards app to get started.
- ✅ Gain access to ready-made dashboards.
- ❌ Gain access to out-of-the-box dashboards.

### Platform components
- **AppEngine** (not "Application Framework")
- **AppShell** — browser runtime isolating apps
- **Launcher** — application to run Dynatrace Apps
- **Dock** — left-hand panel of the Launcher
