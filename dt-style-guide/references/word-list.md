# Dynatrace word list

General English usage rules for Dynatrace content. Not a glossary — a guide to correct
usage of words that appear frequently in Dynatrace writing.
Source: styleguide.dynatrace.com/docs/word-list

---

## Forbidden words

These words must never appear in Dynatrace content. Always use the listed alternatives.

| Forbidden | Alternatives |
|---|---|
| blacklist | blocklist, exclusions, exclude list, denylist, reject list |
| whitelist | allowlist, inclusion list, approved list, safe list |
| master (in tech context) | primary, main, original, source |
| slave | replica, secondary, worker, follower |
| native (referring to people) | Use specific group names |

## Words to avoid or use carefully

| Avoid | Preferred | Reason |
|---|---|---|
| enable / disable | turn on / turn off, allows / activate / deactivate | Ableist connotation |
| e.g. | for example | Spelled out for clarity |
| i.e. | that is | Spelled out for clarity |
| log in / log in to | sign in / sign in to | Dynatrace standard |
| out-of-the-box | ready-made, built-in | |
| navigate to | go to | |
| click | select (device-agnostic) | Unless mouse-specific |
| please | (omit) | Unnecessary in instructions |
| toggle | turn on / turn off | |
| & (ampersand) | and | Except in space-constrained UI |
| 24/7, 24x7 | around the clock | Or specify hours instead; never use "24x7" |

## Common usage rules

### affect vs. effect
- "affect" = verb (to influence): "This might affect performance."
- "effect" = noun (result): "The change had a positive effect."

### after (not "once")
- ✅ After you install the software…
- ❌ Once you install the software…

### although (not "while" for contrast)
- ✅ Although many tools are available…
- ❌ While many tools are available…

### because (not "since" or "as" for causality)
- ✅ The data was lost because the settings are incorrect.
- ❌ The data was lost since the settings are incorrect.

### can / may / might
- "can" = ability: "You can install the software."
- "might" = possibility: "The system might fail."
- "may" = permission only: "You may now proceed."

### compose vs. comprise
- "comprise" = contain: "The platform comprises three tiers."
- "compose" = make up: "Three tiers compose the platform."
- ❌ "comprised of" (always wrong)

### data (singular)
- ✅ The data is overwhelming.
- ❌ The data are overwhelming.

### ensure vs. insure
- "ensure" = make certain
- "insure" = financial protection

### fewer vs. less
- "fewer" = countable items: "fewer hosts"
- "less" = uncountable quantities: "less bandwidth"

### that vs. which
- "that" = restrictive (no comma): "The service that failed…"
- "which" = non-restrictive (with comma): "The service, which failed,…"
